# ikiw — LLM Wiki 지식 베이스

LLM 기반 개인 지식 베이스 관리 skill. 프레임워크, 데이터베이스, 벡터 검색에 의존하지 않으며, 순수 prompt 기반으로 파일 읽기/쓰기가 가능한 모든 LLM agent에서 사용할 수 있습니다.

Karpathy의 LLM Wiki 패턴에서 영감을 받았습니다.

## 명령어

```
ikiw init <path>                    지식 베이스 초기화, 대화를 통해 SCHEMA.md 생성
ikiw summary                        일괄 요약 생성 (미처리 글 감지)
ikiw summary <filename>             지정된 글의 요약 생성
ikiw query "질문"                    지식 베이스 쿼리
ikiw wiki "주제"                     wiki 페이지 생성
ikiw wiki "주제" --style <style>     wiki 생성 후 글쓰기 스타일 적용
ikiw wiki "주제" --design <design>   wiki 생성 후 비주얼 스타일로 HTML 출력
ikiw write "주제" --style <style>    지식 베이스 내용 기반 스타일 글쓰기
ikiw export "주제" --format png      wiki 페이지를 전체 스크린샷으로 내보내기
ikiw export "주제" --format pdf      wiki 페이지를 PDF로 내보내기
ikiw ingest                         새 글 처리 (요약 생성, wiki 업데이트 확인)
ikiw setup-summary                  요약 어시스턴트, 대화를 통해 요약 prompt 정의
```

자연어 트리거도 지원합니다:
- "지식 베이스 검색", "찾아봐", "관련 글 있어?"
- "요약 생성해줘", "새 글 처리해줘"
- "wiki 페이지 만들어줘", "이 주제 정리해줘"
- "지식 베이스에서 XX 관련 내용"

## 핵심 개념

**지식 베이스 = 하나의 디렉토리, 세 가지 요소로 구성:**

```
my-wiki/
├── raw/              # 원본 글, 수정하지 않음
├── summaries.md      # 요약 인덱스, 시스템의 핵심
├── wiki/             # 쿼리 결과 캐시, 필요 시 생성
└── SCHEMA.md         # 이 지식 베이스의 설정
```

- `raw/`는 원본 자료이며, 읽기 전용으로 절대 수정하지 않음
- `summaries.md`는 LLM의 글로벌 맵으로, 각 글마다 하나의 요약이 있음
- `wiki/`는 필요에 따라 생성되는 지식 페이지이며, 본질적으로 쿼리 캐시
- `SCHEMA.md`는 이 지식 베이스의 규칙과 커스텀 prompt를 정의

## 기능

### 1. 요약 생성

raw/에 있는 글을 읽고, SCHEMA.md에 정의된 요약 prompt에 따라 요약을 생성하여 summaries.md에 추가합니다.

**단일 글 처리:**
1. 글 내용 읽기
2. 요약 prompt에 따라 요약 생성
3. summaries.md 끝에 추가

**일괄 처리:**
1. raw/의 모든 파일 목록 조회
2. summaries.md에 이미 있는 파일명과 비교하여 미처리된 글 찾기
3. 글마다 순차적으로 요약을 생성하여 추가
4. 가장 효율적인 처리 방식을 스스로 판단 (직접 순차 처리, 스크립트 병렬 처리, 배치 처리 등)

### 2. 쿼리

사용자가 구체적인 의도를 가지고 질문하면, 지식 베이스에서 답을 찾습니다.

**흐름:**
1. summaries.md 읽기 (전체 요약 인덱스)
2. 질문을 기반으로 요약에서 관련 글 판단
3. 관련 원문 읽기 (보통 10-30편)
4. 종합적으로 답변하며 구체적인 글 인용
5. 답변에 장기적 재사용 가치가 있다면 wiki 페이지로 저장 제안

### 3. Wiki 생성

특정 주제의 지식을 하나의 구조화된 페이지로 종합합니다.

**흐름:**
1. summaries.md를 읽고 관련 글 선별
2. 관련 원문 읽기
3. SCHEMA.md에 정의된 wiki prompt에 따라 페이지 생성
4. 프레임워크 방식으로 구성: 각 지식 포인트 = 한 줄 요약 + 원문 링크
5. wiki/ 디렉토리에 저장

**Wiki는 캐시이지 권위 있는 문서가 아닙니다:**
- 만료될 수 있고, 재생성할 수 있음
- 새 글이 들어오면 관련 wiki가 업데이트 필요할 수 있음
- 완벽함이 아닌 유용함을 추구

### 4. 스타일 출력 (선택 사항)

기본 출력은 markdown입니다. 사용자가 더 나은 시각적 표현이 필요한 경우, design-md 스타일과 결합하여 styled HTML을 생성할 수 있습니다.

**사용 방법:**
- 사용자가 스타일 지정: "Notion 스타일로 출력해줘", "Claude 스타일로 웹페이지 생성해줘"
- `designs/` 디렉토리에서 해당 스타일 파일 읽기
- 스타일 규칙에 따라 wiki 내용을 styled HTML 페이지로 렌더링
- HTML 파일을 지식 베이스의 `wiki/` 디렉토리에 저장 (.md와 같은 이름, 확장자 .html)

**내장 스타일:** 이 skill의 `designs/` 디렉토리에 위치
- `notion` — 따뜻한 미니멀, 세리프 제목, 부드러운 표면
- `claude` — 테라코타 톤, 에디토리얼 레이아웃, 서적 느낌
- `linear.app` — 울트라 미니멀, 정밀함, 퍼플 포인트
- `stripe` — 시그니처 퍼플 그라데이션, 가볍고 우아함
- `vercel` — 블랙 & 화이트 정밀함, Geist 폰트
- `mintlify` — 그린 포인트, 읽기 최적화

**추가 스타일:** https://github.com/VoltAgent/awesome-design-md
- 설치: `npx getdesign@latest add <style-name>`
- 생성된 DESIGN.md를 이름을 변경하여 `designs/` 디렉토리에 배치

### 5. 스타일 글쓰기

지식 베이스 내용을 기반으로, 지정된 글쓰기 스타일 템플릿에 따라 글을 출력합니다.

**흐름:**
1. 콘텐츠 소스 가져오기 (wiki 페이지, 쿼리 결과, 또는 지정된 원문)
2. `styles/` 디렉토리에서 해당 글쓰기 스타일 템플릿 읽기
3. 스타일 템플릿에 따라 내용 리라이팅
4. markdown으로 출력하며, design-md를 중첩 적용하여 styled HTML 생성 가능

**내장 글쓰기 스타일:** 이 skill의 `styles/` 디렉토리에 위치
- 사용자가 직접 스타일 템플릿을 추가하여 `styles/` 디렉토리에 넣을 수 있음

**세 가지 레이어 조합 가능:**
- 콘텐츠 레이어 (wiki prompt) → 무엇을 쓸지 결정
- 스타일 레이어 (styles/) → 어떻게 쓸지 결정
- 비주얼 레이어 (designs/) → 어떻게 보여줄지 결정

사용자가 "샤오홍슈 쇼핑 wiki를 공중하오 스타일로 글을 쓰고, Notion 스타일로 웹페이지를 생성해줘"라고 하면, agent는 wiki 내용, 글쓰기 스타일 템플릿, 비주얼 스타일을 차례로 읽어 레이어별로 렌더링하여 출력합니다.

### 6. 내보내기

wiki 페이지 또는 styled HTML을 이미지나 PDF로 내보냅니다.

**흐름:**
1. 내보낼 콘텐츠 확인 (wiki 페이지, styled HTML)
2. markdown만 있는 경우, 먼저 지정된 design으로 styled HTML을 생성
3. Playwright를 사용하여 전체 페이지 스크린샷으로 내보내기:
   - PNG 전체 길이 이미지: `npx playwright screenshot --full-page input.html output.png`
   - PDF: Playwright의 PDF 내보내기 기능을 통해 생성
4. 출력 파일을 지식 베이스의 `wiki/` 디렉토리에 저장

**지원 형식:**
- `png` — 전체 페이지 스크린샷, 소셜 미디어 공유에 적합
- `pdf` — 인쇄 및 보관에 적합

**조합 사용 예시:**
- `ikiw export "주제" --format png` — 기존 wiki 페이지를 전체 길이 이미지로 내보내기
- `ikiw wiki "주제" --design notion` 후 `ikiw export "주제" --format png` — 먼저 스타일 페이지를 생성한 후 내보내기

### 7. 새 글 입고

새 글이 raw/에 추가된 후의 처리 흐름:
1. raw/에서 summaries.md에 없는 글 감지
2. 요약 생성 후 summaries.md에 추가
3. wiki/에서 업데이트가 필요할 수 있는 관련 페이지가 있는지 확인하고 사용자에게 알림

## 새 지식 베이스 초기화

사용자가 "지식 베이스를 만들어줘"라고 하면:

1. 지식 베이스 디렉토리 경로 확인
2. 디렉토리 구조 생성: raw/, wiki/
3. 대화를 통해 사용자의 글 유형과 관심사 파악
4. 대화 기반으로 커스텀 SCHEMA.md 생성 (요약 prompt, wiki prompt)
5. 빈 summaries.md 생성
6. 사용자에게 글을 raw/에 넣도록 안내
7. 요약 생성 시작

## 요약 어시스턴트

처음 사용 시, 대화를 통해 사용자에게 적합한 요약 prompt를 정의합니다:

- "글의 유형이 무엇인가요?" (기술 블로그, 비즈니스 사례, 학술 논문...)
- "검색할 때 가장 중요한 정보는 무엇인가요?" (방법론, 데이터, 인물, 타임라인...)
- "요약 길이는 어느 정도가 적당한가요?" (1-2문장 초간결, 3-5문장 표준, 한 문단 상세)

답변을 기반으로 요약 prompt를 생성하여 SCHEMA.md에 기록합니다. 사용자는 언제든지 조정할 수 있습니다.

## 다중 지식 베이스 지원

하나의 skill로 여러 지식 베이스를 관리할 수 있으며, 각각 자체 SCHEMA.md를 가집니다:

```
~/wikis/재테크/SCHEMA.md
~/wikis/트레이딩전략/SCHEMA.md
~/wikis/기술노트/SCHEMA.md
```

쿼리 시 지식 베이스를 지정하거나, agent가 질문에 따라 어느 지식 베이스를 검색할지 자동으로 판단합니다.

## 규모 한계

- **1만 편 이내:** summaries.md를 한 번에 읽을 수 있으며, 추가 인프라 불필요
- **1만 편 초과:** 벡터 검색을 도입하여 대략적 필터링 필요, 요약 데이터를 직접 embedding에 활용 가능
- 초대규모를 미리 설계하지 말 것, 필요할 때 추가
