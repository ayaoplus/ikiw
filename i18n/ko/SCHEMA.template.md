# 지식 베이스 설정

이 파일은 이 지식 베이스의 규칙을 정의합니다. LLM agent는 이 지식 베이스를 조작하기 전에 이 파일을 먼저 읽어야 합니다.

## 기본 정보

- **이름:** {{知识库名称}}
- **설명:** {{一句话描述这个知识库的用途}}
- **디렉토리:** {{知识库根目录的绝对路径}}

## 디렉토리 구조

```
{{知识库根目录}}/
├── raw/              # 원본 글, 읽기 전용
├── summaries.md      # 요약 인덱스
├── wiki/             # 필요에 따라 생성되는 지식 페이지
└── SCHEMA.md         # 이 파일
```

## 요약 Prompt

요약을 생성할 때 아래 prompt를 사용합니다. 각 글마다 하나의 요약을 생성하여 summaries.md에 추가합니다.

```
다음 글을 읽고 3-5문장의 요약을 생성하세요. 요약에는 다음 내용이 포함되어야 합니다:
- 저자가 누구이며 어떤 배경을 가지고 있는지
- 핵심적으로 무엇을 했거나 어떤 방법을 공유했는지
- 주요 데이터 (수익, 성장률, 전환율 등, 있는 경우)
- 어떤 플랫폼이나 분야가 관련되는지

요약은 원문을 읽지 않고도 해당 글이 특정 쿼리와 관련이 있는지 판단할 수 있어야 합니다.
```

> 위는 기본 prompt입니다. 사용자는 자신의 글 유형과 관심 분야에 맞게 커스터마이징할 수 있습니다.

## summaries.md 형식

각 항목은 파일명 제목과 요약 본문으로 구성됩니다:

```markdown
## {{文件名}}.md
{{3-5문장 요약}}

## {{文件名}}.md
{{3-5문장 요약}}
```

- 파일명이 곧 링크이며, Obsidian에서 `[[raw/文件名]]`으로 이동할 수 있습니다
- 새 글은 파일 끝에 추가합니다
- 파일을 분리하지 않고 항상 하나의 파일로 유지합니다

## Wiki Prompt

wiki 페이지를 생성할 때 아래 prompt를 사용합니다:

```
다음 글들을 바탕으로 "{{主题名}}" 주제의 wiki 페이지를 생성하세요.

요구사항:
- 체계적으로 구성하고 계층 구조를 명확히 하세요
- 각 지식 포인트 = 한 문장 요약 + 원문 링크 ([[raw/文件名]] 형식 사용)
- 여러 글의 공통점과 차이점을 종합하세요, 단순 나열은 피하세요
- 원문을 그대로 복사하지 마세요
- 글마다 상충되는 관점이 있다면 명확히 지적하세요
```

> 위는 기본 prompt입니다. 사용자는 wiki 페이지의 구조와 스타일을 커스터마이징할 수 있습니다.

## 쿼리 워크플로우

1. summaries.md 읽기
2. 요약을 바탕으로 질문과 관련된 글 판별
3. 관련 원문 읽기
4. 구체적인 글을 인용하여 종합적으로 답변
5. 장기적 가치가 있다면 wiki 페이지로 저장 제안

## 스타일 출력 (선택사항)

기본 출력은 markdown입니다. 스타일이 적용된 HTML 출력이 필요한 경우:

- 스타일 파일은 skill 디렉토리의 `designs/` 하위 디렉토리에 위치합니다
- 내장 스타일: notion, claude, linear.app, stripe, vercel, mintlify
- 사용자가 "XX 스타일로 출력해줘"라고 하면 해당 스타일 파일을 읽어 styled HTML로 렌더링합니다
- HTML 파일은 자체 완결형 단일 파일입니다 (인라인 CSS, 외부 의존성 없음)

**스타일 추가:** https://github.com/VoltAgent/awesome-design-md
- 설치: `npx getdesign@latest add <style-name>`
- 생성된 DESIGN.md의 이름을 변경한 후 skill의 `designs/` 디렉토리에 배치하세요
