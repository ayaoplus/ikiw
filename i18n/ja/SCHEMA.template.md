# ナレッジベース設定

このファイルはナレッジベースのルールを定義します。LLM agent はナレッジベースを操作する前にこのファイルを読み込む必要があります。

## 基本情報

- **名前:** {{知识库名称}}
- **説明:** {{一句话描述这个知识库的用途}}
- **ディレクトリ:** {{知识库根目录的绝对路径}}

## ディレクトリ構成

```
{{知识库根目录}}/
├── raw/              # 元記事、読み取り専用
├── summaries.md      # 要約インデックス
├── wiki/             # 必要に応じて生成されるナレッジページ
└── SCHEMA.md         # このファイル
```

## 要約 Prompt

要約を生成する際は以下の prompt を使用します。各記事につき1つの要約を生成し、summaries.md に追記します。

```
以下の記事を読み、3〜5文の要約を生成してください。要約には以下を含めてください：
- 著者は誰か、どのような経歴か
- 核心的に何をしたか、どのような手法を共有したか
- 主要なデータ（収益、成長率、コンバージョン率など、ある場合）
- どのプラットフォームや分野に関わるか

要約は、原文を読まなくても、その記事が特定のクエリに関連するかどうか判断できるものにしてください。
```

> 上記はデフォルトの prompt です。ユーザーは記事の種類や関心分野に合わせてカスタマイズできます。

## summaries.md の形式

各エントリはファイル名の見出しと要約本文で構成されます：

```markdown
## {{文件名}}.md
{{3〜5文の要約}}

## {{文件名}}.md
{{3〜5文の要約}}
```

- ファイル名がそのままリンクとなり、Obsidian では `[[raw/文件名]]` で遷移できます
- 新しい記事はファイル末尾に追記します
- ファイルを分割せず、常に1つのファイルとして管理します

## Wiki Prompt

wiki ページを生成する際は以下の prompt を使用します：

```
以下の記事をもとに、「{{主题名}}」というテーマの wiki ページを生成してください。

要件：
- フレームワーク型で整理し、階層を明確にする
- 各ナレッジポイント = 一文の要約 + 原文へのリンク（[[raw/文件名]] 形式を使用）
- 複数記事の共通点と相違点を総合する、単なる羅列はしない
- 原文を大量にコピーしない
- 記事間で矛盾する見解がある場合は明確に指摘する
```

> 上記はデフォルトの prompt です。ユーザーは wiki ページの構成やスタイルをカスタマイズできます。

## クエリワークフロー

1. summaries.md を読み込む
2. 要約から質問に関連する記事を判定する
3. 関連する原文を読み込む
4. 具体的な記事を引用しながら総合的に回答する
5. 長期的な価値がある場合は wiki ページとして保存を提案する

## スタイル出力（オプション）

デフォルト出力は markdown です。スタイル付き HTML 出力が必要な場合：

- スタイルファイルは skill ディレクトリの `designs/` サブディレクトリにあります
- 組み込みスタイル：notion、claude、linear.app、stripe、vercel、mintlify
- ユーザーが「XX スタイルで出力して」と言った場合、対応するスタイルファイルを読み込み、styled HTML としてレンダリングします
- HTML ファイルはインライン CSS を含む自己完結型の単一ファイルです（外部依存なし）

**スタイルの追加：** https://github.com/VoltAgent/awesome-design-md
- インストール：`npx getdesign@latest add <style-name>`
- 生成された DESIGN.md をリネームして skill の `designs/` ディレクトリに配置してください
