# 插件开发约定

ikiw 的插件是一个 `.md` 文件，放到 `plugins/` 目录即可生效。agent 在读取 SKILL.md 后会扫描此目录，按需加载。

## 插件文件结构

每个插件 `.md` 文件必须包含以下部分：

```markdown
# 插件名称

> 一句话描述

## 命令

定义该插件提供的命令及参数。

## 触发词

自然语言触发方式。

## 依赖

该插件依赖的资源目录或外部工具。

## 执行流程

分步描述 agent 应如何执行该插件的功能。
```

## 命名约定

- 文件名用英文，`kebab-case` 格式
- 名称应能直接表达功能，如 `screenshot-export.md`、`poster-generator.md`
- 避免模糊命名，如 `export.md`、`output.md`

## 资源引用

插件可引用以下共享资源目录：

| 目录 | 内容 | 说明 |
|------|------|------|
| `designs/` | 视觉样式文件 | 每个 `.md` 文件是一个 design-md 样式定义 |
| `styles/` | 写作风格模板 | 每个 `.md` 文件是一个写作风格定义 |
| `templates/` | 结构化模板 | 如海报骨架模板，定义内容提炼规则和 HTML 结构 |

## 编写原则

- **只写"做什么"，不写"怎么做"** — agent 自己决定实现方式
- **但要写清楚关键约束** — 如默认参数、降级策略、输出路径
- **实现细节可以写** — 当且仅当不写的话 agent 会做错（如 Playwright 的 file:// 协议、deviceScaleFactor 参数）
- **保持精简** — 一个插件文件控制在 100 行以内

## 示例

参考官方插件：
- `screenshot-export.md` — 截图导出
- `poster-generator.md` — 海报生成
- `style-writer.md` — 风格写作
