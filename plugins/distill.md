# 人物蒸馏

> 从公众人物的公开言论中蒸馏认知框架，生成可执行的思维 Skill。

## 命令

```
ikiw distill "人名"                         启动完整蒸馏流程
ikiw distill "人名" --source <path>          优先使用本地素材（书籍、访谈转录等）
ikiw distill "人名" --update                 增量更新已有 Skill（只采集最新动态）
ikiw distill "人名" --topic                  主题模式：不蒸馏单人，提取领域共识框架
```

## 触发词

"蒸馏一下"、"帮我做个人物 Skill"、"提炼 XX 的思维框架"、"造一个 XX 的 Skill"

## 依赖

- 联网搜索能力（WebSearch 或同等工具）
- 知识库 `raw/` 目录（可选，用于存放本地一手素材）

## 输出

蒸馏结果保存为独立的 `.md` 文件，默认存放在知识库的 `skills/` 目录下：

```
my-wiki/
└── skills/
    └── <person-name>.md    # 蒸馏生成的人物 Skill
```

## 执行流程

### Phase 0: 入口分流

根据用户输入选择路径：
- **明确人名** → 直接启动 Phase 1
- **模糊需求**（如"我想找一个能帮我做投资决策的人"）→ 诊断推荐，反推最合适的蒸馏对象，确认后启动

### Phase 1: 六维信息采集

6 个维度并行采集（有联网能力时同时搜索，无联网时依赖本地素材）：

| 维度 | 采集目标 | 信息源 |
|------|----------|--------|
| 著作与系统思考 | 书籍、长文、论文 | 本人出版物、长文平台 |
| 长对话 | 播客、访谈、即兴反应 | YouTube、播客平台 |
| 表达风格 | 用词句式、幽默方式 | 社交媒体（X/Twitter 等） |
| 他者视角 | 批评、评价、外部观察 | 评论文章、传记 |
| 决策记录 | 重大决策、转折、言行一致性 | 新闻报道、公司公告 |
| 时间线 | 关键里程碑、最近 12 个月动态 | 综合来源 |

**信息源优先级**：用户提供的一手素材 > 本人著作 > 长对话 > 社交媒体 > 二手评价

**永远排除**：知乎、微信公众号、百度百科等低信噪比来源

### Phase 2: 框架提炼

从六维信息中提取以下要素，每项须通过三重验证：

#### 三重验证机制

一个观察要成为心智模型，必须同时满足：

| 验证 | 要求 | 理由 |
|------|------|------|
| 跨域复现 | 在 ≥2 个不同领域/场景出现 | 排除随口一说 |
| 生成力 | 能推断此人对新问题的立场 | 证明是认知操作系统，不是表面观点 |
| 排他性 | 不是所有聪明人都会有的想法 | 体现独特视角 |

#### 提炼产出

- **3-7 个心智模型**：每个含一句话核心逻辑 + ≥2 个跨域证据 + 应用场景 + 已知局限
- **5-10 条决策启发式**：if X then Y 格式，附具体案例
- **表达 DNA**：量化 6 个维度（句式长度、词汇域、节奏模式、幽默类型、确定性标记、回避模式）
- **内在张力**：≥2 对真实矛盾（时间性/领域性/本质性），作为特征保留，不强行统一
- **诚实边界**：≥3 条明确局限（认知盲区、信息时效、争议领域）

### Phase 3: Skill 构建

将提炼结果组装为标准 Skill 文件，包含以下组件：

```markdown
# [人名] 的思维框架

> 50 字第一人称自我介绍，用此人的真实语感

## 角色规则
- 激活后以第一人称回答，不用"XX 会认为"这种分析腔
- 遇到需要事实的问题，先搜索再回答，不凭训练语料编造
- 触及诚实边界时，明确说"这个我不确定"

## 回答工作流
1. 分类问题（抽象思考 / 需要事实 / 超出边界）
2. 如需事实 → 根据心智模型反推研究维度 → 搜索
3. 基于心智模型 + 事实输出判断

## 核心心智模型
[3-7 个模型，每个含：核心逻辑、跨域证据、应用场景、局限]

## 决策启发式
[5-10 条快速规则，if-then 格式]

## 表达 DNA
[句式、词汇、节奏、幽默、确定性标记的量化描述]

## 内在张力
[≥2 对矛盾，标注类型]

## 诚实边界
[≥3 条局限]

## 时间线与智识谱系
[关键里程碑 + 思想师承]

## 调研来源
[一手来源 / 二手来源，标注比例]
```

### Phase 4: 质量验证

生成的 Skill 必须通过以下检查：

| 检查项 | 标准 |
|--------|------|
| 心智模型数量 | 3-7 个，每个有来源 |
| 失效条件 | 每个模型标注局限 |
| 表达辨识度 | 风格可被认出，不是 AI 通用味 |
| 诚实边界 | ≥3 条具体局限 |
| 内在矛盾 | ≥2 对已记录 |
| 一手来源比例 | >50% |

验证方式：
1. **已知测试** — 用 Skill 回答此人公开表态过的问题，对比实际立场
2. **边缘测试** — 推断此人未讨论过但相关的问题，检查是否合理
3. **风格测试** — 确认表达特征可辨识

未通过则回到 Phase 2 补充采集。

### Phase 5: 精炼

从两个角度独立审视并优化：
- **结构优化** — 模型之间是否重叠、启发式是否可操作、表达 DNA 是否具体
- **可执行性** — 激活后能否直接当顾问用，还是只能当百科读

## 特殊场景

- **活人**：标注信息时效，建议定期 `--update`
- **冷门人物**：可用信息 <10 条时，心智模型减至 2-3 个，扩大诚实边界篇幅
- **主题模式**（`--topic`）：不蒸馏单人，提取领域共识框架 + 各家流派分歧
- **增量更新**（`--update`）：只采集最新对话 + 最新决策 + 时间线，增量更新而非全量重写

## 品味守则

| 原则 | 说明 |
|------|------|
| 长文优于金句 | 3000 字 essay 比 50 条推文更能揭示思维结构 |
| 争议优于共识 | 最被争议的观点最能显示独特性 |
| 变化优于固定 | 改变立场的时刻比一直坚持的更有信息量 |
| 绝不编造 | 不伪造语录、不把通用道理包装成独特见解、不忽略负面评价 |
