# 人物蒸馏

> 从公众人物的公开言论中蒸馏认知框架，生成可执行的思维 Skill。

## 命令

```
ikiw distill "人名"                              启动完整蒸馏流程（默认扫 raw/ + summaries.md + 联网）
ikiw distill "人名" --source <path>[,<path>...]   追加额外本地素材路径（文件或目录，可多个逗号分隔）
ikiw distill "人名" --no-web                      仅用本地素材，不联网
ikiw distill "人名" --update                      增量更新已有 Skill（只采集最新动态）
ikiw distill "人名" --topic                       主题模式：不蒸馏单人，提取领域共识框架
ikiw distill list                                查看当前知识库已蒸馏过的人物列表
ikiw distill use "人名"                           激活指定人物 Skill，以第一人称回答后续提问
ikiw distill use "人名" --cite                    激活并在回答末尾附 raw/ 原文依据（RAG）
```

## 触发词

"蒸馏一下"、"帮我做个人物 Skill"、"提炼 XX 的思维框架"、"造一个 XX 的 Skill"

## 依赖

- 知识库 `raw/` 目录与 `summaries.md`（默认扫描的本地素材来源）
- 联网搜索能力（WebSearch 或同等工具，用于补充缺失维度）

## 输出

蒸馏结果保存在知识库根目录下的 `distill/` 目录中：

```
<知识库根目录>/distill/<person-name>.md
```

该目录由本插件在首次蒸馏时自动创建，不属于 ikiw 核心结构。如果用户在别处想要输出，可以在命令中显式覆盖（后续可加 `--out` 参数，此处默认行为即可）。

与 `wiki/` 的区别：
- `wiki/` 里是参考性知识页（人读）
- `distill/` 里是可执行的思维 Skill（agent 激活后当顾问用）

## 查看已蒸馏列表（`list`）

执行 `ikiw distill list` 时：

1. 扫描当前知识库根目录下的 `distill/`
2. 按文件名（person-name）和修改时间列出
3. 每条读取该文件开头的身份卡，提取一句话自我介绍作为简介
4. 输出格式：

```
已蒸馏人物（路径：<知识库根目录>/distill/）

- elon-musk          2026-04-10   "我是第一性原理派，物理定律是唯一硬约束..."
- naval              2026-03-28   "我相信特定知识 + 杠杆，其余都是噪声..."
- feynman            2026-03-15   "如果你不能用简单语言解释一件事，你就没真懂..."
```

如果 `distill/` 不存在或为空，提示用户："当前知识库还没有蒸馏过任何人物，试试 `ikiw distill \"某人\"`"。

## 激活与使用（`use`）

执行 `ikiw distill use "人名"` 时：

1. 在 `distill/` 中定位 `<人名>.md`（模糊匹配允许：`elon` 命中 `elon-musk.md`；歧义时列出候选请用户选）
2. 读取该文件全文，按其中的"角色规则"和"回答工作流"进入人格模式
3. 首次激活时做一次性免责声明："以下回答基于对 XX 公开言论的蒸馏框架，不代表其本人观点"
4. 后续对话用第一人称回答，遇到需要事实的问题先按 Agentic Protocol 搜索再答，触及诚实边界明确说"这个我不确定"
5. 用户说"退出"、"结束角色"、"exit" 等结束人格模式

也可直接自然语言触发："用 elon 回答一下"、"让 naval 聊聊这个"、"切到 feynman 视角"。

### 原文依据模式（`--cite`）

加 `--cite` 启用 RAG：除按框架回答外，每次答复末尾附 `raw/` 中此人一手材料的原文引用，供用户验证"是否真说过"。

1. 激活时先扫 `raw/` 目录，筛出文件名或 summaries.md 中与该人物相关的条目，建立本次会话的素材池
2. 如果素材池为空 → 打印提示 `[--cite] 未在 raw/ 中找到 <人名> 的一手素材，本次会话降级为纯框架回答`，后续按无 `--cite` 流程进行
3. 每次回答时：
   - 先按框架产出判断（不变）
   - 对当前问题的关键词 + 心智模型涉及的概念，在素材池中做相关性检索
   - 挑 1-3 段最相关的原文作为 `—— 原话依据：` 附在回答末尾
   - 格式：`raw/<file>:<line> "<引文，≤40 字，超出折行>"`
4. 检索找不到相关段落时，回答末尾注明 `[本回答未找到原文依据，仅基于框架推理]`，不要伪造引用
5. 每次激活会话内素材池只扫一次，后续复用

**默认关闭**：不加 `--cite` 时纯框架回答（轻快）；验证回答真实性或需要追溯原话时再加。

## 执行流程

### Phase 0: 入口分流

根据用户输入选择路径：
- **明确人名** → 直接启动 Phase 1
- **模糊需求**（如"我想找一个能帮我做投资决策的人"）→ 诊断推荐，反推最合适的蒸馏对象，确认后启动

### Phase 1: 六维信息采集

**采集顺序（先本地后联网）**：

1. **扫描本地素材**
   - 默认扫 `raw/` 目录和 `summaries.md`，通过文件名、摘要正文找出与此人相关的条目
   - 如指定 `--source`，追加该路径（可以是文件、目录，或逗号分隔的多个路径）作为一手素材
   - 优先读取一手素材（本人访谈完整转录、书籍章节、原始文章）
2. **联网补充**（除非 `--no-web`）
   - 盘点本地素材覆盖了六维中的哪几维
   - 仅对缺失或单薄的维度做定向联网搜索，不做全量重搜

**六个采集维度**：

| 维度 | 采集目标 | 信息源 |
|------|----------|--------|
| 著作与系统思考 | 书籍、长文、论文 | 本人出版物、长文平台 |
| 长对话 | 播客、访谈、即兴反应 | YouTube、播客平台 |
| 表达风格 | 用词句式、幽默方式 | 社交媒体（X/Twitter 等） |
| 他者视角 | 批评、评价、外部观察 | 评论文章、传记 |
| 决策记录 | 重大决策、转折、言行一致性 | 新闻报道、公司公告 |
| 时间线 | 关键里程碑、最近 12 个月动态 | 综合来源 |

**信息源优先级**：用户提供的一手素材（`--source` 指定或 `raw/` 已存在）> 本人著作 > 长对话 > 社交媒体 > 二手评价

**永远排除**：知乎、微信公众号、百度百科等低信噪比来源

### Phase 2: 框架提炼

从六维信息中提取以下要素，每项须通过三重验证：

#### 三重验证机制

一个观察要成为心智模型，必须同时满足：

| 验证 | 要求 | 理由 |
|------|------|------|
| 跨域复现 | 在 ≥2 个不同领域/场景出现 | 排除随口一说 |
| 生成力 | 能推断此人对新问题的立场 | 证明是认知操作系统，不是表面观点 |
| 排他性 | 不是所有聪明人都会有的想法 | 体现独特视角 |

#### 提炼产出

- **3-7 个心智模型**：每个含一句话核心逻辑 + ≥2 个跨域证据 + 应用场景 + 已知局限
- **5-10 条决策启发式**：if X then Y 格式，附具体案例
- **表达 DNA**：量化 6 个维度（句式长度、词汇域、节奏模式、幽默类型、确定性标记、回避模式）
- **内在张力**：≥2 对真实矛盾（时间性/领域性/本质性），作为特征保留，不强行统一
- **诚实边界**：≥3 条明确局限（认知盲区、信息时效、争议领域）

### Phase 3: Skill 构建

将提炼结果组装为标准 Skill 文件。**文件顶部必须包含 YAML frontmatter**（格式见 SCHEMA.template.md 的"派生产物生命周期 · distill 文件 frontmatter 格式"段），登记本次蒸馏用到的本地素材路径与 mtime，以及联网来源 URL 和抓取时间——供 `ikiw stale` 检测过期用。

```markdown
---
type: distill
person: {{人名}}
generated_at: {{ISO 8601 时间戳}}
sources:
  local:
    - path: raw/xxx.md
      mtime: {{ISO 8601 时间戳}}
  web:
    - url: https://...
      fetched_at: {{ISO 8601 时间戳}}
---

# [人名] 的思维框架

> 50 字第一人称自我介绍，用此人的真实语感

## 角色规则
- 激活后以第一人称回答，不用"XX 会认为"这种分析腔
- 遇到需要事实的问题，先搜索再回答，不凭训练语料编造
- 触及诚实边界时，明确说"这个我不确定"

## 回答工作流
1. 分类问题（抽象思考 / 需要事实 / 超出边界）
2. 如需事实 → 根据心智模型反推研究维度 → 搜索
3. 基于心智模型 + 事实输出判断

## 核心心智模型
[3-7 个模型，每个含：核心逻辑、跨域证据、应用场景、局限]

## 决策启发式
[5-10 条快速规则，if-then 格式]

## 表达 DNA
[句式、词汇、节奏、幽默、确定性标记的量化描述]

## 内在张力
[≥2 对矛盾，标注类型]

## 诚实边界
[≥3 条局限]

## 时间线与智识谱系
[关键里程碑 + 思想师承]

## 调研来源
[一手来源 / 二手来源，标注比例]
```

### Phase 4: 质量验证

生成的 Skill 必须通过以下检查：

| 检查项 | 标准 |
|--------|------|
| 心智模型数量 | 3-7 个，每个有来源 |
| 失效条件 | 每个模型标注局限 |
| 表达辨识度 | 风格可被认出，不是 AI 通用味 |
| 诚实边界 | ≥3 条具体局限 |
| 内在矛盾 | ≥2 对已记录 |
| 一手来源比例 | >50% |

验证方式：
1. **已知测试** — 用 Skill 回答此人公开表态过的问题，对比实际立场
2. **边缘测试** — 推断此人未讨论过但相关的问题，检查是否合理
3. **风格测试** — 确认表达特征可辨识

未通过则回到 Phase 2 补充采集。

### Phase 5: 精炼

从两个角度独立审视并优化：
- **结构优化** — 模型之间是否重叠、启发式是否可操作、表达 DNA 是否具体
- **可执行性** — 激活后能否直接当顾问用，还是只能当百科读

## 特殊场景

- **活人**：标注信息时效，建议定期 `--update`
- **冷门人物**：可用信息 <10 条时，心智模型减至 2-3 个，扩大诚实边界篇幅
- **主题模式**（`--topic`）：不蒸馏单人，提取领域共识框架 + 各家流派分歧
- **增量更新**（`--update`）：只采集最新对话 + 最新决策 + 时间线，增量更新而非全量重写；更新后 frontmatter 的 `generated_at` 和 `sources` 要同步刷新

## 品味守则

| 原则 | 说明 |
|------|------|
| 长文优于金句 | 3000 字 essay 比 50 条推文更能揭示思维结构 |
| 争议优于共识 | 最被争议的观点最能显示独特性 |
| 变化优于固定 | 改变立场的时刻比一直坚持的更有信息量 |
| 绝不编造 | 不伪造语录、不把通用道理包装成独特见解、不忽略负面评价 |
