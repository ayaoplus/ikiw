# 风格写作

> 基于知识库内容，按指定的写作风格模板输出文章。

## 命令

```
ikiw write "主题" --style <style>                  按风格模板写作
ikiw write "主题" --style <style> --design <design> 写作 + 视觉样式输出 HTML
```

## 触发词

"帮我写篇文章"、"用 XX 风格写"、"改写成公众号体"

## 依赖

- `styles/` — 写作风格模板
- `designs/` — 视觉样式文件（可选，叠加使用）

## 执行流程

1. 获取内容来源（wiki 页面、查询结果、或指定的原文）
2. 读取 `styles/<style>.md` 获取写作风格定义
3. 按风格模板重写内容
4. 默认输出 markdown
5. 如指定 `--design`，再叠加视觉样式生成 styled HTML

## 三层可组合

- **内容层**（SCHEMA.md 的 wiki prompt）→ 决定写什么
- **风格层**（`styles/`）→ 决定怎么写
- **视觉层**（`designs/`）→ 决定怎么呈现

示例："把小红书带货的 wiki 用公众号风格写篇文章，用 Notion 样式生成网页"
→ agent 依次读取 wiki 内容、写作风格模板、视觉样式，逐层渲染。

## 自定义风格

用户将写作风格模板放入 `styles/` 目录即可。
风格模板定义语气、结构、遣词造句的规则，格式不限，agent 能读懂即可。
